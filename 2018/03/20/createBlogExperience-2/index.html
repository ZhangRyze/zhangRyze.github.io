写在开始之前

自从上回折腾完博客 “不了了之” 之后时隔一年第二次折腾，是因为现在想写一些东西但是看着原来的那个主题不太满意在搜索浏览的过程中发现了 Hexo，比 Jeklly 功能差不多，然后就试着用 Hexo 重新搭建了博客。现在来分享一下过程。

Hexo

Hexo 是一个快速、简洁且高效的博客框架。同样使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

安装 Hexo 相当简单，只需要用到两个工具：

  • node.js
  • Git
安装 Git
  • Windows平台：下载并安装 Git。
  • Mac平台：使用 Homebrew, MacPorts：brew install git ；或下载 安装程序 安装。
安装 node.js

安装 Node.js 的最佳方式是使用 nvm。

  • 另外，您也可以使用 Git Bash，这是 git for windows 自带的一组程序，提供了 Linux 风格的 shell，在该环境下，您可以直接用上面提到的命令来安装Node.js。打开它的方法很简单，在任意位置单击右键，选择 Git Bash Here 即可。由于 Hexo 的很多操作都涉及到命令行，您可以考虑始终使用 Git Bash 来进行操作。

文件夹 scaffolds

模版 的文件夹，当您新建文章时，Hexo 会根据 scaffold 来建立文件。Hexo 的模板是指在新建的 markdown 文件中默认填充的内容。例如：

新建 post 的命令，修改scaffold/post.md 中的 Front-matter 内容，那么每次新建一篇文章时都会包含这个修改。

  • 新建 page 的命令，修改scaffold/page.md 中的 Front-matter 内容，那么每次新建一篇文章时都会包含这个修改。
  • 新建scaffold/xxxx.md 文件，那么每次执行 $ hexo new xxxx 'PageName' 新建一篇文章时都会包含这个修改。
  • Front-matter 是用 --- 包含的内容，博客渲染完是不显示在界面上，只存在该页面的参数里
文件夹 source

资源文件夹是存放用户资源的地方，Markdown(.md) 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

除 _posts 文件夹之外，开头命名为 _ (下划线)的文件/文件夹和隐藏的文件将会被忽略。

文件夹 themes

主题 文件夹。Hexo 会根据主题来生成静态页面。会有一个默认的主题文件夹
landscape，landscape 的默认语法是 ejs。

  • 主题的文件夹里也有个 _config.yml 文件，通常在配置一些跟主题相关的信息，例如：menu（菜单）等，当然在主文件夹的 _config.yml 中配置也可以，在这里主要是为了区别主题配置和网站配置。

  • 主题文件夹中的 language 文件夹可以配置网站的语言，然后在主配置 _config.yml 文件中的 language 填入您想要显示的 language 文件夹中的语言文件名。在布局过程中使用 __(value) 方法转换您的关键词。

    • 语言仅限一些您在布局过程中网站的关键词。
    • 上述 value 是您在语言文件中的 key 值，__(value) 得到的是它对应的值。

  • 主题文件夹中的 layout 文件夹是放整个博客的布局的文件夹，用于存放主题的模板文件，决定了网站内容的呈现方式，Hexo 内建 Swig 模板引擎，您可以另外安装插件来获得 EJS、Haml 或 Jade（Pug） 支持，Hexo 的默认主题 landscape 是使用 Ejs。Hexo 根据模板文件的扩展名来决定所使用的模板引擎，例如：

  • 主题文件夹中的 source 文件夹是资源文件夹，除了模板以外的 Asset，例如 CSS、JavaScript 文件等，都应该放在这个文件夹中。文件或文件夹开头名称为 _（下划线线）或隐藏的文件会被忽略。如果文件可以被渲染的话，会经过解析然后储存到 public 文件夹，否则会直接拷贝到 public 文件夹。
